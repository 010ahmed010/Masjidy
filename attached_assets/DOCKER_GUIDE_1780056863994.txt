IBN GROUP - Docker Deployment Guide
=====================================

REQUIREMENTS
------------
- Docker installed on your VPS
- A MongoDB Atlas connection string (Atlas must whitelist your server IP or use 0.0.0.0/0)
- A strong JWT secret key (generate it once and save it safely)


GENERATE YOUR JWT SECRET (do this once)
----------------------------------------
Run this command on any machine that has Node.js:

    node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

Copy the output and save it in a safe place (password manager, etc.).
You MUST use this same value every time you run or restart the container.


BUILD THE DOCKER IMAGE
-----------------------
Upload your project files to the VPS, then run:

    docker build -t ibn-group:v1.0 .

To rebuild after code changes:

    docker build --no-cache -t ibn-group:v1.0 .


RECOMMENDED: USE AN ENV FILE (more secure than inline flags)
-------------------------------------------------------------
Create a file at /etc/ibn-group.env on your server (readable only by root):

    MONGODB_URI=your_mongodb_connection_string
    JWT_SECRET=your_saved_jwt_secret
    ADMIN_PASSWORD=your_admin_password
    ADMIN_ID=                        <- the admin account's database ID (same type as member IDs).
                                        the admin account is auto-created on first startup, so this
                                        ID does not exist yet before the first run.
                                        workflow: leave blank → run container → get the ID using
                                        the steps below → add it here → restart once → done forever.
                                        this value never changes, you only do this one time.
    NODE_ENV=production
    PORT=8000

Secure the file:

    chmod 600 /etc/ibn-group.env


RUN THE CONTAINER
------------------
Using env file (recommended):

    docker run -d \
      --restart unless-stopped \
      --env-file /etc/ibn-group.env \
      -p 80:8000 \
      --name ibn-group \
      ibn-group:v1.0

Using inline flags (quick testing only — secrets are visible in process list):

    docker run -d \
      --restart unless-stopped \
      -p 80:8000 \
      -e MONGODB_URI=your_mongodb_connection_string \
      -e JWT_SECRET=your_saved_jwt_secret \
      -e ADMIN_PASSWORD=your_admin_password \
      -e ADMIN_ID= \
      --name ibn-group \
      ibn-group:v1.0

NOTE: --restart unless-stopped means the container automatically restarts
if it crashes or the server reboots. Always include this flag in production.


HOW TO GET YOUR ADMIN_ID (required for payment notifications)
--------------------------------------------------------------
When a member submits a withdrawal or deposit request, the server sends a
notification to the admin account. This requires knowing the admin's database ID.

Steps:
1. Start the container WITHOUT ADMIN_ID first (leave it blank).
2. Log in at your site as admin.
3. On the server, run:

       docker exec ibn-group node -e "
         require('dotenv').config();
         const mongoose = require('mongoose');
         mongoose.connect(process.env.MONGODB_URI).then(async () => {
           const User = require('./models/User');
           const admin = await User.findOne({ role: 'admin' });
           console.log('ADMIN_ID:', admin._id.toString());
           process.exit(0);
         });
       "

4. Copy the printed ID and set ADMIN_ID= in your env file.
5. Restart the container:

       docker stop ibn-group && docker rm ibn-group
       docker run -d --restart unless-stopped --env-file /etc/ibn-group.env -p 80:8000 --name ibn-group ibn-group:v1.0


IMPORTANT - JWT SECRET RULE
-----------------------------
The JWT secret is used to sign and verify user login tokens.

- Every time you restart or redeploy the container, you MUST use
  the exact same JWT_SECRET value.

- If you use a different secret after restart, all currently logged-in
  users will be logged out and must log in again.

- If you forget to pass JWT_SECRET, the server will refuse to start
  and show a FATAL error. This is intentional for security.


IMPORTANT - MONGODB CONNECTION
--------------------------------
- In production, if the MongoDB Atlas connection fails at startup, the server
  will exit immediately with a FATAL error instead of falling back to any
  local database. This prevents silent data loss.

- Make sure your Atlas cluster has your VPS IP address whitelisted, or use
  0.0.0.0/0 (allow all) for simplicity.

- Test your connection string before deploying:

      docker run --rm -e MONGODB_URI=your_string ibn-group:v1.0 \
        node -e "require('./config/db')()"


DEFAULT ADMIN LOGIN
--------------------
Username : admin
Password : the value you set for ADMIN_PASSWORD
           (defaults to Admin@IBN2024 if not set)


ENVIRONMENT VARIABLES SUMMARY
-------------------------------
MONGODB_URI      - (required) Your MongoDB Atlas connection string
JWT_SECRET       - (required) Your saved JWT secret key — never change this
ADMIN_PASSWORD   - (optional) Admin account password, default: Admin@IBN2024
ADMIN_ID         - (recommended) Admin user's MongoDB _id for payment notifications
PORT             - (optional) Internal server port, default: 8000
NODE_ENV         - Set automatically to "production" by the Dockerfile


USEFUL DOCKER COMMANDS
-----------------------
View live logs:
    docker logs -f ibn-group

View last 100 lines of logs:
    docker logs --tail 100 ibn-group

Stop the container:
    docker stop ibn-group

Start a stopped container (preserves env variables):
    docker start ibn-group

Remove the container (to run a fresh one):
    docker rm -f ibn-group

Check if the container is running:
    docker ps

Check resource usage:
    docker stats ibn-group

Health check (test the API is responding):
    curl http://localhost/api/health


UPGRADING TO A NEW VERSION
----------------------------
1. Upload the updated project files to the VPS.
2. Build the new image:
       docker build --no-cache -t ibn-group:v1.1 .
3. Stop and remove the old container:
       docker rm -f ibn-group
4. Start the new container (same env file, new image tag):
       docker run -d --restart unless-stopped --env-file /etc/ibn-group.env -p 80:8000 --name ibn-group ibn-group:v1.1
